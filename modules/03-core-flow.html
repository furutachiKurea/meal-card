代码翻译：消费核心逻辑

CODE 
err = s.cardRepo.DB().Transaction(func(tx *gorm.DB) error {
    txRepo := s.cardRepo.WithTx(tx)
    card, err := txRepo.FindCardByCardNo(cardNo)
    if err != nil {
        return newBizError(ErrCodeCardNotFound, "此卡非本单位所发")
    }
    if card.Status == model.CardStatusCancelled {
        return newBizError(ErrCodeCardCancelled, "此卡已注销")
    }
    if card.Status == model.CardStatusLost {
        return newBizError(ErrCodeCardLost, "此卡已挂失")
    }
    card.Balance -= amount
    txRepo.UpdateCard(card)
    txRepo.CreateTransaction(txn)
    return nil
})
中文解释

Transaction(func...) — 开启事务

WithTx(tx) — 创建绑定本次事务的仓库实例

FindCardByCardNo — 查卡，找不到 = 不是我们发的

Status == Cancelled — 已注销，拒绝

Status == Lost — 已挂失，拒绝并报警

Balance -= amount — 扣减余额（单位：分）

UpdateCard + CreateTransaction — 更新余额 + 写流水，事务保证不可分割

return nil — 成功，事务提交

两个关键设计思想

💡
事务：要么全成功，要么全回滚

扣款成功但写流水断电？有事务保护 → 扣款自动撤销 → 余额恢复。

🏗️
三层架构：各管各的

Handler 只管 HTTP · Service 只管业务规则 · Repository 只管数据库。改一层不影响其他层。
