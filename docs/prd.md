# 产品需求文档（PRD）

## 系统角色

- **管理处操作员**：在管理端执行发卡、存款、挂失、取消挂失、注销、查看统计报表
- **窗口工作人员**：在窗口端执行就餐消费操作（刷卡、输入金额、结算）

## 功能 1：发卡

**触发**：持卡人持有效证件到管理处。

**流程**：
1. 操作员录入持卡人信息（姓名、证件号）
2. 操作员录入押金金额
3. 操作员录入预存款金额（可为 0）
4. 若该证件号已有挂失中的卡，系统自动将旧卡注销（退还旧卡押金和余额）
5. 系统创建新卡，自动生成卡号（每次发卡都是新卡新编号，不重用旧卡号）
6. 系统记录押金、预存款，卡状态为 active

**约束**：
- 一人一卡：同一证件号只能持有一张 active 状态的卡
- 已有 active 卡时不能办新卡
- 已有 lost 卡时可以办新卡，旧卡自动注销
- 押金必须大于 0
- 预存款金额 >= 0

## 功能 2：存款

**触发**：持卡人到管理处充值。

**流程**：
1. 操作员输入卡号
2. 系统校验卡存在且状态为 active
3. 操作员输入充值金额
4. 系统更新余额，生成存款记录
5. 可打印收据

**收据内容**：卡号、持卡人姓名、充值金额、充值后余额、充值时间。

**约束**：
- 充值金额必须大于 0
- 已挂失或已注销的卡不能充值

## 功能 3：就餐消费

**触发**：持卡人在窗口刷卡。

**流程**：
1. 工作人员输入卡号（模拟刷卡）
2. 系统执行三重校验：
   - 卡号是否存在（是否为本单位所发）
   - 卡是否有效（状态非 cancelled）
   - 卡是否已挂失（状态非 lost）
3. 校验失败 → 页面报警提示，说明失败原因
4. 校验通过 → 页面显示卡内余额
5. 工作人员输入本次消费金额 → 页面显示消费金额
6. 工作人员确认结算 → 系统扣款，生成消费记录
7. 页面显示扣款后的新余额

**页面结构**：窗口端是工作人员操作的界面，同时展示持卡人可见的信息（余额、消费金额、新余额）。

**约束**：
- 消费金额必须大于 0
- 余额不足时拒绝结算
- 每笔消费必须关联窗口

## 功能 4：汇总统计

**触发**：管理处操作员查看报表。

**统计项**：
- 本餐售饭总收入（按时间范围筛选）
- 各窗口收入（按窗口分组）
- 各持卡人存款明细（按持卡人列出每笔存款）
- 本日存款金额、本月存款金额
- 卡中流动资金总额（所有 active 卡的余额之和）
- 日餐报表（按日汇总消费数据）
- 年餐报表（按年汇总消费数据）

**约束**：
- 所有统计基于已有的消费记录和存款记录聚合查询，不需要额外的统计表

## 功能 5：挂失 / 取消挂失

### 挂失

**触发**：持卡人到管理处报告卡丢失。

**流程**：
1. 操作员输入卡号
2. 系统校验卡存在且状态为 active
3. 系统将卡状态改为 lost
4. 挂失后，该卡在任何窗口刷卡都会触发报警（功能 3 的校验环节拦截）

### 取消挂失

**触发**：持卡人找回卡后到管理处取消挂失。

**流程**：
1. 操作员输入卡号
2. 系统校验卡存在且状态为 lost
3. 系统将卡状态恢复为 active

**约束**：
- 只有 active 的卡可以挂失
- 只有 lost 的卡可以取消挂失

## 功能 6：注销

**触发**：持卡人不再就餐，到管理处注销。

**流程**：
1. 操作员输入卡号
2. 系统校验卡存在且状态为 active 或 lost
3. 系统将卡状态改为 cancelled
4. 退还押金和卡内余额（余额清零）
5. 页面显示应退金额（押金 + 余额）

**约束**：
- 已注销的卡不能再注销
- 注销后该卡号永久停用，不可重新发卡（每次发卡都是新编号）
