# 发卡

## 当前行为

操作员录入证件号，系统调学籍验证服务校验身份，展示姓名和人员类型供操作员二次确认，确认后录入预存款，系统自动创建新卡。

## 前置条件

- 证件号必须存在于学籍验证服务（12 位数字学号/工号）
- 该证件号当前没有 active 状态的卡

## 成功路径

1. 操作员输入 12 位证件号
2. 系统调学籍验证服务，返回姓名和人员类型（学生/教职工）
3. 前端展示验证结果，操作员二次确认
4. 操作员录入预存款金额（可为 0）
5. 系统创建新卡：押金固定 20 元（2000 分）由系统写入，生成 16 位数字卡号，状态 active

## 异常 / 边界情况

- 证件号不在学籍库：校验失败，流程中止，不创建卡
- 已有 active 卡：返回错误，不创建新卡
- 已有 lost 卡：系统自动注销旧卡（余额清零、退还旧押金+余额），再创建新卡，响应中包含旧卡退款信息

## 规则 / 限制

- 押金固定 2000 分（20 元），不可由操作员调整，DB 仍存储以便退款时使用
- 卡号为 16 位随机数字字符串，独立于数据库自增 ID，每次发卡生成新号不重用
- 一人一卡：同一证件号最多持有一张 active 卡
- 预存款 >= 0
