# 挂失 / 取消挂失

## 当前行为

操作员输入证件号，系统查找该证件号名下当前有效卡并展示，操作员确认后执行挂失或取消挂失。

## 前置条件（挂失）

- 证件号名下存在 active 状态的卡

## 前置条件（取消挂失）

- 证件号名下存在 lost 状态的卡

## 成功路径（挂失）

1. 操作员输入 12 位证件号
2. 系统按证件号查找当前有效卡（active 或 lost），展示卡号、姓名、状态
3. 操作员确认
4. 系统将卡状态改为 lost

## 成功路径（取消挂失）

1. 操作员输入 12 位证件号
2. 系统按证件号查找当前有效卡，展示卡信息
3. 操作员确认
4. 系统将卡状态从 lost 恢复为 active

## 异常 / 边界情况

- 证件号无有效卡（只有 cancelled 或无记录）：返回"无有效卡"，流程中止
- 对 lost 状态的卡执行挂失：返回错误，只有 active 卡可挂失
- 对 active/cancelled 状态的卡执行取消挂失：返回错误，只有 lost 卡可取消挂失

## 规则 / 限制

- 操作入口为证件号（学号/工号），不直接输入卡号
- 挂失后该卡在任何窗口刷卡均触发报警拦截
