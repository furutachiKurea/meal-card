# 窗口机双屏协同

## 当前行为
- 窗口机拆分为两个独立页面：操作端（`/window`）和顾客屏（`/window/customer`）
- 操作端由食堂工作人员使用，负责刷卡验证、输入消费金额、确认结算
- 顾客屏面向学生，只读展示余额、消费金额、结算结果、报警信息
- 两端通过浏览器 BroadcastChannel API 实时同步状态

## 前置条件
- 操作端和顾客屏必须在同一浏览器实例的不同标签页中打开（同源限制）
- 两端的 URL 参数 `?id=N` 必须一致，才能连接到同一频道

## 成功路径
1. 工作人员在操作端刷卡 → 顾客屏自动显示持卡人姓名和余额
2. 工作人员输入金额并确认结算 → 顾客屏显示本次消费和剩余余额
3. 工作人员点击"下一位" → 顾客屏回到"请刷卡"等待状态

## 异常 / 边界情况
- 刷卡验证失败（挂失/注销/非本单位卡）→ 操作端显示警报，同时顾客屏显示红色报警
- 顾客屏未打开或频道不匹配 → 操作端正常工作不受影响，消息丢失无副作用
- 多个顾客屏监听同一频道 → 所有顾客屏都会收到状态更新

## 规则 / 限制
- 频道命名：有 `?id=N` 时为 `window-{N}`，无参数时为 `window-default`
- 消息类型：`card_read`（刷卡成功）、`settled`（结算完成）、`alarm`（报警）、`reset`（重置）
- 顾客屏完全只读，不调用任何后端接口
- 操作端只发送不接收
