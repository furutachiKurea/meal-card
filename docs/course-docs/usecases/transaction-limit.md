# 消费限额

## 当前行为
- 每次就餐消费时，系统自动校验单笔限额和日累计限额
- 超限时拒绝结算，返回明确错误信息

## 前置条件
- 卡状态为 active
- 窗口有效

## 成功路径
1. 消费金额 ≤ 200 元（单笔限额）
2. 今日已消费 + 本次金额 ≤ 500 元（日累计限额）
3. 余额充足
4. 正常扣款

## 异常 / 边界情况
- 单笔超过 200 元 → 返回 `EXCEED_SINGLE_LIMIT`，HTTP 403
- 今日累计将超过 500 元 → 返回 `EXCEED_DAILY_LIMIT`，HTTP 403
- 日限额计算范围：当天 00:00:00 到次日 00:00:00（服务器本地时区）
- 跨天后限额重置（新的一天从 0 开始累计）

## 规则 / 限制
- 单笔上限：200 元（20000 分），常量 `MaxSingleTransaction`
- 日累计上限：500 元（50000 分），常量 `MaxDailyTransaction`
- 限额校验在余额校验之前执行
- 单笔限额在事务外校验（纯数值比较），日限额在事务内校验（需查询当天消费总额）
