# 存款

## 当前行为
- 操作员输入 16 位卡号查询卡信息，确认卡状态为 active 后输入充值金额
- 系统更新余额并生成存款记录，返回收据信息
- 收据包含：单号、卡号、持卡人姓名、充值金额、充值后余额、充值时间
- 支持打印收据（浏览器 window.print，打印区域隔离不影响其他页面元素）

## 前置条件
- 持卡人持有 active 状态的饭卡
- 操作员知道持卡人的 16 位卡号

## 成功路径
1. 操作员输入 16 位卡号，点击查询
2. 系统展示持卡人姓名和当前余额
3. 操作员输入充值金额（> 0）
4. 点击"确认存款"
5. 系统扣款成功，页面展示收据（含打印按钮）

## 异常 / 边界情况
- 卡号不存在 → 提示"卡号不存在"
- 卡已挂失 → 提示"该卡已挂失，无法充值"
- 卡已注销 → 提示"该卡已注销，无法充值"
- 充值金额 <= 0 → 前端表单校验拒绝提交

## 规则 / 限制
- 充值金额必须大于 0
- 只有 active 状态的卡可以充值，lost 和 cancelled 均拒绝
- 存款操作在数据库事务内完成（余额更新 + 存款记录创建原子性）
- 收据 ID 为后端自增 DepositRecord.ID
