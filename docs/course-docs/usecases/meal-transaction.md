# 就餐消费

## 当前行为
- 窗口工作人员通过窗口机页面为持卡人刷卡结算就餐费用
- 刷卡后系统执行三重校验（卡存在、非注销、非挂失），通过后显示余额
- 工作人员输入消费金额并确认结算，系统扣款并显示新余额

## 前置条件
- 至少存在一个窗口（通过管理端窗口管理页面创建）
- 持卡人持有 active 状态的饭卡

## 成功路径
1. 工作人员输入 16 位卡号（模拟刷卡）
2. 系统展示持卡人姓名和当前余额
3. 工作人员输入消费金额
4. 点击"确认结算"
5. 系统扣款成功，页面展示本次消费金额和扣款后余额
6. 点击"下一位"重置页面，准备为下一位持卡人服务

## 异常 / 边界情况
- 卡号不存在 → 警报"此卡非本单位所发，禁止就餐"
- 卡已注销 → 警报"此卡已注销，禁止就餐"
- 卡已挂失 → 警报"此卡已挂失，禁止就餐"
- 余额不足 → 提示"余额不足"，拒绝结算
- 消费金额 <= 0 → 前端表单校验拒绝提交

## 规则 / 限制
- 消费金额必须大于 0
- 每笔消费必须关联一个窗口
- 三重校验顺序严格：卡存在 → 非 cancelled → 非 lost → 余额充足
- 校验失败时页面以醒目红色警报展示，工作人员只能点"重新刷卡"

## 窗口绑定
- 窗口机通过 URL 参数 `/window?id=1` 绑定到指定窗口
- 绑定后窗口选择器隐藏，所有消费自动关联该窗口
- 不带参数时显示窗口选择器，默认选中第一个窗口
- 真实部署时每台窗口机配置固定 URL，避免工作人员误选窗口
