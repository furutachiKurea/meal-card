# card_holders（持卡人）

## 作用
- 记录办卡人的身份信息，作为饭卡的归属主体

## 设计原因
- 持卡人和饭卡是一对多关系（一个人可能因注销再办卡等场景持有不同卡），需要独立存储身份信息
- 证件号设唯一索引，防止同一人重复登记

## 字段
- ID:
  - 含义: 持卡人主键
  - 类型: uint
  - 是否必填: 自动生成
  - 默认值: 自增
  - 备注: GORM primaryKey
- Name:
  - 含义: 姓名
  - 类型: string
  - 是否必填: 是
  - 默认值: 无
- IDNumber:
  - 含义: 证件号（身份证/学生证等）
  - 类型: string
  - 是否必填: 是
  - 默认值: 无
  - 备注: 唯一索引
- CreatedAt:
  - 含义: 登记时间
  - 类型: time.Time
  - 是否必填: GORM 自动填充
  - 默认值: 当前时间

## 主键 / 索引 / 约束
- 主键: ID
- 唯一索引: IDNumber
- NOT NULL: Name, IDNumber

## 表关系
- 被 cards 表通过 card_holder_id 引用（一对多）
