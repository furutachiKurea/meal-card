# IssuePage（发卡页）

## 作用
- 管理员为新持卡人办理就餐卡，分两步：先验证证件号身份，再录入预存款提交发卡

## 结构
- 单 Card 布局，最大宽 520px，居中
- 步骤一：证件号输入区 + 验证结果展示区（姓名、人员类型）+ 确认按钮
- 步骤二：预存款输入表单 + 提交按钮（仅在身份验证通过且操作员确认后显示）
- 结果区（下方，条件渲染）：发卡成功信息卡片，可能包含旧卡注销退款子卡片

## 用户操作
1. 输入 12 位证件号（学号/工号），点击「验证」
   - 验证失败（非本校人员）：展示错误 Alert
   - 验证成功：展示姓名和人员类型，显示「确认」按钮
2. 操作员确认身份后点击「确认，继续办卡」
3. 录入预存款金额（可选，≥ 0），点击「办理发卡」提交
4. 发卡成功：展示卡号（16位）、持卡人、押金（20元，系统固定）、余额
5. 若旧卡自动注销：在成功卡片下方额外展示旧卡退款明细（押金/余额/合计）

## 变更说明（相对旧版）
- **移除**：姓名输入框（姓名从学籍验证结果获取）
- **移除**：押金输入框（系统固定 20 元，前端不可修改）
- **新增**：证件号验证步骤（调 GET /api/validate-student）
- **新增**：验证结果展示区 + 二次确认按钮

## 状态变化
```
初始
→ 输入证件号，点验证 → loading
  → 验证失败: error 填入
  → 验证成功: studentInfo 填入，展示确认区
→ 操作员点「确认」→ confirmed=true，展示预存款表单
→ 提交发卡 → loading
  → 成功: result 填入，状态重置到初始（可继续办下一张）
  → 失败: error 填入
```

## 接口依赖
- `GET /api/validate-student?idNumber=xxx`（validateStudent）
  - 成功返回：`{ valid, idNumber, name, type }`
- `POST /api/cards`（issueCard）
  - 入参：idNumber, preDeposit（分）
  - 不再传入 name 和 deposit
  - 金额处理：显示元，提交前 × 100 转分

## 备注 / 待确认
- 若同证件号已有 lost 卡，后端自动注销旧卡，响应含 `refund` 字段，前端判断 `result.refund` 是否存在决定是否渲染退款块
- 发卡成功后展示的押金固定为 20.00 元（从 `result.card.deposit / 100` 得到，值恒为 2000 分）
