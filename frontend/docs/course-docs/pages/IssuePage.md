# IssuePage（发卡页）

## 作用
- 管理员为新持卡人办理就餐卡，填写持卡人信息、押金和预存金额后提交发卡

## 结构
- 单 Card 布局，最大宽 520px，居中
- 表单区（上方）：4 个字段
- 结果区（下方，条件渲染）：发卡成功信息卡片，可能包含旧卡注销退款子卡片

## 用户操作
- 填写持卡人姓名（必填）、证件号（必填）、押金（必填，>0.01）、预存金额（可选，≥0）
- 点击「办理发卡」提交
- 发卡成功后：表单重置，展示卡号、持卡人、押金、余额
- 若旧卡自动注销：在成功卡片下方额外展示旧卡退款明细（押金/余额/合计）

## 状态变化
```
初始 → loading=true（提交中）→ 成功: result 填入，form 重置
                              → 失败: error 填入
```

## 接口依赖
- `POST /api/cards`（issueCard）
  - 入参：name, idNumber, deposit（分）, preDeposit（分）
  - 金额处理：页面显示元，提交前 × 100 转分

## 备注 / 待确认
- 若同一证件号已有活跃卡，后端会自动注销旧卡并在响应中返回 `refund` 字段，前端判断 `result.refund` 是否存在来决定是否渲染退款块
- 押金最小值 0.01 元，预存金额可为 0
