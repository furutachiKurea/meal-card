# 前端

React + Vite + React Router

## 环境要求

- Node.js 18+
- pnpm

## 启动

```bash
# 安装依赖
pnpm install

# 开发模式（访问 http://localhost:5173）
pnpm dev

# 构建产物
pnpm build
```

> 启动前请确保后端服务已在 `http://localhost:8080` 运行。

## 页面说明

### 发卡 `/issue`

为新持卡人办理就餐卡。

填写持卡人姓名、证件号、押金和预存金额后提交，成功后显示生成的卡号。

- 同一证件号只能持有一张有效卡，重复办卡会提示冲突

### 存款 `/deposit`

为已有卡充值。

1. 输入卡号，点击「查询」显示持卡人和当前余额
2. 输入充值金额，点击「存款」
3. 成功后展示收据（卡号、持卡人、充值金额、最新余额、时间）

### 就餐消费 `/meal`

模拟窗口机刷卡消费流程。

1. 输入卡号，点击「读卡」
   - 卡正常：显示持卡人姓名和余额
   - 卡挂失/注销/不存在：显示红色报警提示，流程终止
2. 输入本次消费金额，点击「结算」
3. 成功后显示扣款后的剩余余额

### 挂失管理 `/loss`

挂失或取消挂失就餐卡。

输入卡号查询后，根据当前状态显示对应操作按钮：
- 卡状态为「正常」：可执行挂失
- 卡状态为「已挂失」：可执行取消挂失

### 注销 `/cancel`

注销就餐卡并退款。

1. 输入卡号查询，确认持卡人信息
2. 点击「注销」后需二次确认
3. 成功后展示退款明细（押金退还 + 余额退还）

### 汇总统计 `/statistics`

管理处查看各类统计数据，共 7 个区块：

| 区块 | 说明 | 筛选条件 |
|---|---|---|
| 售饭总收入 | 指定日期范围内的消费总金额 | 开始/结束日期 |
| 各窗口收入 | 每个窗口的消费汇总 | 无 |
| 持卡人存款明细 | 每位持卡人的充值记录 | 无 |
| 本日/月存款 | 今日和本月的充值总额 | 无 |
| 流动资金总额 | 当前所有有效卡的余额之和 | 无 |
| 日餐报表 | 指定日期的消费流水 | 日期 |
| 年餐报表 | 指定年份按月汇总的消费数据 | 年份 |

### 窗口管理 `/windows`

查看和创建消费窗口。

- 页面加载时自动展示所有窗口列表
- 填写窗口名称后可新建窗口
