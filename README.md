# 食堂饭卡管理系统

课设项目，前后端分离架构，实现发卡、存款、就餐消费、挂失、注销、汇总统计 6 项功能。

交互式代码讲解：[meal-card-course/index.html](meal-card-course/index.html)

## 环境准备

分别参考各子目录 README 配置运行环境：

- 后端：[backend/README.md](backend/README.md)（需要 Go 1.26+）
- 前端：[frontend/README.md](frontend/README.md)（需要 Node.js 18+、pnpm 8+）

## 启动

```bash
# 后端（监听 :8080）
make dev-backend

# 前端（监听 :5173）
make dev-frontend

# 运行后端单测
make test
```

分两个终端分别运行 `dev-backend` 和 `dev-frontend`。

### 完整流程验收步骤

按以下顺序逐步验证，每步通过后再进行下一步。

#### 1. 发卡

1. 进入「发卡」页面
2. 输入证件号（如 `202100010001`），点击「验证」，应显示姓名"张三"
3. 填写预存金额（如 `50.00`），点击「发卡」
4. 应看到发卡成功提示，并展示生成的卡号（16 位）
5. 记录该卡号，后续步骤使用

#### 2. 存款

1. 进入「存款」页面
2. 输入上一步得到的卡号，点击「查询」，应显示当前余额（等于预存金额）
3. 填写存款金额（如 `30.00`），点击「存款」
4. 应提示存款成功，并可打印收据（收据上应显示单号、卡号、金额、时间）
5. 再次查询该卡，余额应增加 30.00 元

#### 3. 就餐消费

1. 进入「就餐」页面，选择一个窗口
2. 输入卡号，点击「刷卡」，应通过验证并显示持卡人姓名和当前余额
3. 输入消费金额（如 `8.50`），点击「确认结算」
4. 应显示结算成功，本次消费金额和扣款后余额均正确

#### 4. 挂失

1. 进入「挂失 / 解挂」页面
2. 输入证件号（12 位），点击查询，确认卡状态为正常
3. 点击「挂失」，应提示挂失成功，卡状态变为已挂失
4. 回到就餐页，用同一张卡刷卡，应触发警报"此卡已挂失，禁止就餐"
5. 回到挂失页，对同一张卡点击「解挂」，状态应恢复正常
6. 再次在就餐页刷卡，应正常通过验证

#### 5. 注销

1. 进入「注销」页面
2. 输入证件号（12 位），点击查询，确认卡信息正常
3. 点击「注销」，应提示注销成功
4. 在就餐页用已注销卡刷卡，应触发警报"此卡已注销，禁止就餐"
5. 在存款页查询已注销卡，应提示卡状态异常或无法存款

#### 6. 汇总统计

1. 进入「统计」页面
2. **本餐售饭总收入**：选择时间范围覆盖上述操作时间，点击查询，收入应与第 3 步消费金额一致
3. **各窗口收入**：同上时间范围，应能看到对应窗口的收入明细（表格支持分页，每页 10 条）
4. **各持卡人存款明细**：不选时间范围直接查询，应能看到第 2 步的存款记录（外层每页 10 人，内层每页 10 条）
5. **本日/本月存款金额**：点击查询，今日存款应包含第 2 步的金额
6. **日餐报表**：选择今天，应能看到第 3 步的消费笔数和收入
7. **年餐报表**：选择当前年份，应能看到本月有消费数据
